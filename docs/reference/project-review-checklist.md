# 👑 キマチェック（言語非依存中核）

## 目的
コード・ドキュメントが**法典（ADR）とルール（CLAUDE.md / .claude/rules/）**に準拠しているか確認し、
**チェックと同時に**、準拠していなければ**全ソース・全ドキュメントを修正**する。

陛下が「キマチェック」と言ったとき、本チェックリストに従って実行せよ。

## 実行方針

1. **チェック即修正**: 違反・齟齬を発見したら、その Phase の対象となる**全ソース・全ドキュメント**を漏れなく修正する。影響する参照・呼び出し元もすべて更新する。
2. **「今後したほうがいい、今はしなくても大丈夫」は修正しない**: そのような事項は**今回の修正対象にしない**。最後に陛下へ**お伝えのみ**する。

---

## Phase 1: 法典準拠（ADR Compliance）

### 1-1. コードがADRに違反していないか
- **対象**: プロジェクトのソースコード全体
- **アクション**: 違反 → **該当する全ソースを修正**

### 1-2. ADRに記録すべき決定が漏れていないか
- **対象**: 前回チェック以降に実装された新機能・アーキテクチャ変更
- **アクション**: 漏れ → **ADR追加**

### 1-3. 既存ADRが陳腐化していないか
- **対象**: `docs/adr/` 配下の全 ADR
- **アクション**: 乖離 → **ADR修正** or **Superseded**

---

## Phase 2: ルール準拠

### 2-1. ディレクトリ方針
- **対象**: ソースコードのファイル配置
- **アクション**: 役割に反する配置 → **ファイル移動**、**参照を更新**

### 2-2. 技術スタック
- **対象**: 依存関係マニフェスト（`package.json`, `pyproject.toml`, `Cargo.toml`, 等）
- **アクション**: 未承認ライブラリ → **ADR追加して正式採用** or **削除**

### 2-3. 品質管理
- **対象**: プロジェクト全体のコード
- **確認基準（言語非依存の共通項目）**:
  - Magic Number（定数で定義すべき数値の直書き）
  - 秘密情報直書き（APIキー等）
  - 未使用のインポート・変数・関数
  - エラーハンドリングの妥当性
- **言語固有の確認基準はプロジェクトの CLAUDE.md で追記すること**（例: Python なら型ヒント・docstring、TypeScript なら strict mode）
- **アクション**: 違反 → **該当する全ソースを修正**

---

## Phase 3: ドキュメント整合

### 3-1. ドキュメント配置ルール
- **対象**: `docs/` 配下の全ドキュメント
- **アクション**: ずれあり → **ファイル移動** or **配置の是正**、**参照パスを修正**

### 3-2. コードとの整合
- **対象**: `docs/` 配下
- **アクション**: 齟齬 → **該当する全ドキュメントを修正**

### 3-3. 参照パス
- **対象**: ドキュメント内のファイルパス参照
- **アクション**: 壊れたリンク → **全パスを修正**

### 3-4. 統計・数値
- **対象**: プロジェクト構造ドキュメント（存在する場合）
- **アクション**: ずれ → **数値を一括更新**

---

## Phase 4: 不要物の削除

### 4-1. 未使用コード
- **対象**: ソースコード・テスト全体
- **アクション**: 不要 → **削除**、**参照を削除・更新**

### 4-2. 未使用依存
- **対象**: 依存関係マニフェスト
- **アクション**: 不要 → **マニフェストから削除**

---

## Phase 5: リファクタリング

### 5-1. 重複コードの統合
### 5-2. 関数/クラスの分割（50行以上の関数、300行以上のクラス）
### 5-3. 命名の改善
### 5-4. パフォーマンス改善

---

## 報告フォーマット

1. **発見した違反・齟齬**（Phase と項目番号を明記）
2. **実施した修正・追加・削除**（ファイル名と概要）
3. **要対応だが未実施**（理由と推奨対応）
4. **今後対応推奨（今回は未修正）**
