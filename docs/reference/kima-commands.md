# 👑 Kima コマンドシステム

## 概要

キマへの指示を簡潔に行うためのコマンドシステム。
ユーザーが `kima` で始まるメッセージを送信すると、このドキュメントに従ってコマンドを実行する。

---

## コマンド一覧

| コマンド | サブコマンドなし（デフォルト） | サブコマンドあり |
|----------|------------------------------|------------------|
| **status** | 現状サマリを表示する（表示のみ） | `kima status update`（統計を更新） |
| **roadmap** | ロードマップを表示する（表示のみ） | `kima roadmap update`（更新）、`kima roadmap next`（次の一手） |
| **commit** | コミットメッセージ案を表示し、y/n 確認のうえでコミット実行 | — |
| **check** | Phase 1〜5 を実行する | — |
| **order** | 指示書を生成する | — |
| **help** | コマンド一覧を表示する | — |

---

## コマンド詳細

### `kima status`
- **目的**: プロジェクトの現状サマリを表示する（読み取りのみ）。
- **実行内容**: roadmap.md の完了状況、git の状態、統計を要約して表示。

### `kima status update`
- **目的**: 統計を再集計し `docs/reference/project-structure.md` を更新する。

### `kima roadmap`
- **目的**: ロードマップを表示する（読み取りのみ）。

### `kima roadmap update`
- **目的**: 陛下の指示に基づき roadmap.md を更新する（y/n 確認）。

### `kima roadmap next`
- **目的**: 次の一手を確認し、その実装に進む。

### `kima commit`
- **目的**: 規約に沿ったコミットメッセージ案を作成し、y/n 確認のうえコミット。
- **実行内容**:
  1. git status / git diff を把握
  2. `docs/reference/commit-message-format.md` に従いメッセージ案を作成
  3. y/n 確認。y ならコミット実行、n なら中止。

### `kima check`
- **目的**: `docs/reference/project-review-checklist.md` に従い Phase 1〜5 を実行する。

### `kima order`
- **目的**: 直前の会話から Agent 用のコピペ可能な指示書を生成する。

### `kima help`
- **目的**: 利用可能なコマンド一覧を表示する。

---

## 補完アシスト

ユーザーが `kima` とだけ入力した場合、利用可能なコマンド一覧を表示する。

## 不明なコマンド

存在しないコマンドが指定された場合: 「そのコマンドは存在しません。kima help でコマンド一覧を確認してください。」
