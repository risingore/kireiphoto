# ADR 0002: Cursor + Claude Code 二重AI基盤の採用

## ステータス
承認済み (Accepted)

## 決定日
{{DATE}}

## コンテキスト（背景と課題）
AI アシスタント（キマ）を開発環境に統合するにあたり、単一のAIツールでは以下の制約があった：
1. **Cursor 単体**: エディタ内の補完・チャットは強力だが、CLI操作・ファイル操作・エージェント協調には限界がある。
2. **Claude Code 単体**: CLI操作・スキル・エージェント機能は豊富だが、エディタ統合（補完・インラインチャット）が弱い。
3. **ルールの分断**: 両ツールが異なるルールファイルを読むため、ペルソナや行動規範の一貫性が課題となる。

## 決定事項 (Decision)
Cursor（エディタ）と Claude Code（CLI拡張）を併用し、同一のペルソナ・ルールを二重管理する。

1. **Cursor**: `.cursorrules` をプロジェクトルートに配置。Composer / Chat が読み込む。
2. **Claude Code**: `CLAUDE.md` + `.claude/` ディレクトリを配置。CLI・スキル・エージェントが読み込む。
3. **同期方針**: ペルソナ定義・コマンド表・品質管理・思考プロトコルは両ファイルに同一内容を記載する。
4. **差分**: プラットフォーム固有の制約（コードブロック表示、ツール使用強制等）のみ差分を許容する。
5. **詳細ガイド**: `docs/guides/cursor-claude-code-setup.md` に運用手順を記載する。

## 結果 (Consequences)
### メリット
- **最適なツールを場面ごとに使い分け**: エディタ作業は Cursor、CLI操作・自動化は Claude Code と役割分担できる。
- **ペルソナの一貫性**: どちらのツールから呼び出しても同じキマが応答する。
- **拡張性**: Claude Code のスキル・エージェント・チーム機能により、高度な自動化が可能。

### デメリット / 注意点
- **二重管理コスト**: `.cursorrules` と `CLAUDE.md` の同期を忘れるとペルソナが乖離する。
- **新規ルール追加時の手間**: 両方に影響するルールは2ファイルを更新する必要がある。
