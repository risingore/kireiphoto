# ADR 0003: キマコマンドシステムの導入

## ステータス
承認済み (Accepted)

## 決定日
{{DATE}}

## コンテキスト（背景と課題）
AI アシスタント（キマ）への指示が自由文のみだと、以下の問題が発生していた：
1. **指示の曖昧さ**: 「コミットして」「状況見せて」等の自由文は解釈のブレが生じやすい。
2. **手順の非標準化**: commit メッセージフォーマットの参照、ロードマップの確認等、定型作業の手順が毎回異なる。
3. **操作の再現性**: 同じ操作を別の会話で行う際、手順が記憶されていない。

## 決定事項 (Decision)
`kima` プレフィックスによるコマンドシステムを導入する。

1. **コマンド体系**: `kima <subcommand>` 形式で統一する。
2. **主要コマンド**: status, roadmap, commit, check, order, help の6系統。
3. **正本**: コマンド仕様は `docs/reference/kima-commands.md` に定義する。
4. **実行基盤**: Claude Code ではスキル（`.claude/skills/kima/SKILL.md`）、Cursor では `.cursorrules` 内のルーティング表で実行する。
5. **自動参照**: 各コマンドは実行時に必要な参照ドキュメント（commit-message-format.md 等）を自動で読み込む。

## 結果 (Consequences)
### メリット
- **指示の明確化**: コマンド名で意図が一意に決まり、解釈のブレがなくなる。
- **手順の標準化**: commit は必ずフォーマットを参照、check は必ず5フェーズを実行、等が保証される。
- **操作性**: `kima help` で一覧を確認でき、学習コストが低い。

### デメリット / 注意点
- **コマンド追加時**: `docs/reference/kima-commands.md`、スキル、`.cursorrules` の3箇所を更新する必要がある。
- **自由文との併用**: コマンド以外の自由文指示も引き続き受け付けるため、コマンドに無い操作との境界が曖昧になる可能性がある。
